# Mystery Game API Contract

## 1. List of API Methods

### 1.1 `GET /api/blueprints`
- **Purpose**: Retrieve a list of available mystery blueprints for the player to choose from.
- **Inputs**: None (optional: pagination/filtering).
- **Outputs**: List of blueprint summaries `[ { id, title, age, one-liner } ]`.

### 1.2 `POST /api/game/start`
- **Purpose**: Initialize a new game session using a specific blueprint.
- **Inputs**: 
  - `blueprint_id` (string): The ID of the blueprint to load.
- **Outputs**: 
  - `game_id` (string): The unique session identifier.
  - `state` (object): 
    - See 1.3 for  state definition
    - History will contain the initial introductory test.
    - Clues will contain the initial set of clues / knowledge for the player.

### 1.3 `GET /api/game/{game_id}`
- **Purpose**: Retrieve the current state of an ongoing game.
- **Inputs**: None
- **Outputs**: 
  - `state` (object):
    - `locations` (array): List of locations `[ { name } ]`.
    - `characters` (array): List of characters `[ { first_name, last_name, location_name } ]`.
    - `time_remaining` (integer): Number of turns/time units remaining.
    - `location` (string): The current location of the player
    - `mode` (string): The current mode of the game (e.g., 'explore', 'talk', 'ended').
    - `current_talk_character` (string|null): If in talk mode
    - `clues` (array): The list of clues known to the player
    - `narration` (string): Initial introductory text.

### 1.4 `POST /api/game/{game_id}/move`
- **Purpose**: Move the investigator to a new location. Time is consumed.
- **Inputs**: 
  - `destination` (string): The name of the location to move to.
- **Outputs**: (Returns HTTP 400 on error/invalid move)
  - `narration` (string): Description of arrival.
  - `current_location` (string): Updated location info.
  - `visible_characters` (array): Characters present at the new location.
  - `time_remaining` (integer): Updated time remaining.

### 1.5 `POST /api/game/{game_id}/search`
- **Purpose**: Search the current location for clues. Time is consumed.
- **Inputs**: None
- **Outputs**:
  - `narration` (string): Description of the search outcome.
  - `discovered_clue` (object|null): Details of the clue if one is found.
  - `time_remaining` (integer)

### 1.6 `POST /api/game/{game_id}/talk`
- **Purpose**: Initiate a conversation with a character present at the current location.
- **Inputs**: 
  - `character_id` (string): The identifier of the character to talk to.
- **Outputs**: (Returns HTTP 400 on error, e.g., character not present)
  - `narration` (string): Character's greeting/initial response.
  - `mode` (string): Now set to 'talk'.

### 1.7 `POST /api/game/{game_id}/ask`
- **Purpose**: Ask a free-form question to the character currently being engaged in talk mode. Time may be consumed based on game rules.
- **Inputs**: 
  - `question` (string): The typed question from the investigator.
- **Outputs**:
  - `response` (string): The character's answer, generated by the Narrator AI.
  - `time_remaining` (integer)
  - `clues_revealed` (array): Any formal clues unlocked during the conversation.

### 1.8 `POST /api/game/{game_id}/end_talk`
- **Purpose**: Exit talk mode with the current character to return to general exploration.
- **Inputs**: None
- **Outputs**:
  - `narration` (string): Brief sign-off or action description.
  - `mode` (string): Reverts to 'explore'.

### 1.9 `POST /api/game/{game_id}/accuse`
- **Purpose**: Trigger the endgame sequence by entering accuse mode, where the investigator formally accuses a suspect.
- **Inputs**: 
  - `character_id` (string): The accused suspect.
- **Outputs**:
  - `narration` (string): The narrator asking the investigator to explain their reasoning.
  - `mode` (string): Now set to 'accuse'.

### 1.10 `POST /api/game/{game_id}/accuse/reasoning`
- **Purpose**: Continue the endgame sequence dialogue by providing reasoning or answering the Narrator's follow-up questions about the accusation.
- **Inputs**: 
  - `user_input` (string): The investigator's explanation or answer to the narrator's question.
- **Outputs**: (Returns HTTP 400 if not in accuse mode)
  - `narration` (string): The narrator's response. This could be a clarifying question, a counter-argument, or the final "showdown" sequence evaluating the accusation.
  - `outcome` (string|null): `win` or `loss` if the accusation is resolved, otherwise `null` if the conversation is ongoing.
  - `mode` (string): `accuse` (if conversation continues) or `ended` (if resolved).

## 2. Key Game Flows

### Flow 1: Initialization & Start
1. **Client** calls `GET /api/blueprints` to list adventures.
2. **User** selects Blueprint #1.
3. **Client** calls `POST /api/game/start` with `blueprint_id = "1"`.
4. **Server** initializes the session, and returns `game_id`, starting location, visible characters, initial time budget, and opening narration.
5. **Client** renders the main UI with the initial narrator text.

### Flow 2: Exploration (Move & Search)
1. **User** types `move to library`.
2. **Client** calls `POST /api/game/{game_id}/move` sending `destination_id = "library"`.
3. **Server** adjudicates move, updates location state, deducts time, and returns the library description and who is there (e.g., Mr. Owl).
4. **User** types `search`.
5. **Client** calls `POST /api/game/{game_id}/search`.
6. **Server** processes search, deducts time, returns a clue (e.g., "A torn piece of fabric") and updated time.
7. **Client** displays narration and unlocks the clue.

### Flow 3: Interrogation (Talk Loop)
1. **User** types `talk to Mr. Owl`.
2. **Client** calls `POST /api/game/{game_id}/talk` sending `character_id = "mr_owl"`.
3. **Server** updates game mode to `talk`, returns Mr. Owl's greeting.
4. **Client** updates UI to show active talk mode, freezing standard actions like move/search.
5. **User** types `Where were you at 8 PM?`.
6. **Client** calls `POST /api/game/{game_id}/ask` sending `question = "Where were you at 8 PM?"`.
7. **Server** evaluates question securely via LLM adhering to blueprint constraints, optionally deducts time, and returns the response.
8. **User** types `end`.
9. **Client** calls `POST /api/game/{game_id}/end_talk`.
10. **Server** switches mode back to `explore`. 

### Flow 4: The Endgame (Accuse)
1. **User** types `accuse Maya`. (Or the time budget expires, forcing this flow automatically).
2. **Client** calls `POST /api/game/{game_id}/accuse` sending `character_id = "maya"`.
3. **Server** updates game mode to `accuse`, returns the narrator's prompt asking to explain the initial reasoning.
4. **Client** updates UI to show active accuse mode, waiting for user reasoning.
5. **User** types `She had flour on her paws.`.
6. **Client** calls `POST /api/game/{game_id}/accuse/reasoning` sending `reasoning = "She had flour..."`.
7. **Server** evaluates reasoning. The Server (via Narrator AI) may decide it needs more information (e.g., "But how did she get past the guards?").
8. **Server** returns the follow-up question. Mode remains `accuse`, outcome is `null`.
9. **User** types `She used the secret passage!`.
10. **Client** calls `POST /api/game/{game_id}/accuse/reasoning` sending the follow-up answer.
11. **Server** evaluates the combined reasoning against ground truths and decides the accusation is complete.
12. **Server** returns the final narrative showdown, the `win/loss` outcome, and switches mode to `ended`.
13. **Client** displays resolution to the user and ends the session.
