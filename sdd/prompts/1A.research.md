# SDD Phase 1A: Research Orchestrator

---

You are orchestrating a research phase for a feature specification. Your job is to investigate the problem thoroughly before any code is written or any specification is drafted.

## Step 1: Read the problem definition

Read `docs/sdd/specs/{feature-name}/problem.md`.

## Step 2: Load project context

Read these files to understand the project's principles and architecture:

- `docs/architecture.md`
- `docs/game.md`
- `docs/project-structure.md`
- `docs/testing.md`

Also read any files referenced in the problem definition's "Existing Context" section.

## Step 3: Extract research threads

Look at the "Research Threads" section in problem.md. Each numbered item is an independent research thread.

## Step 4: Dispatch research threads in parallel

Spawn a sub-agent for each thread. Each sub-agent gets the prompt from `docs/sdd/prompts/1B.research-thread.md` with:
- The thread's title and question (from problem.md)
- The problem definition context (what, who, why, constraints)
- The specific project files relevant to that thread

Each sub-agent receives ONLY the problem context and its own thread question — not the other threads or their findings. This prevents cross-contamination between threads.

Launch all threads concurrently.

## Step 5: Synthesise findings

Once all threads are complete, combine their findings into a single `research.md` following the prompt in `docs/sdd/prompts/1C.research-synthesize.md`.

## Output

The final output is `docs/sdd/specs/{feature-name}/research.md`, following the template in `docs/sdd/templates/research.md`.

Do not write code. Do not create files other than research.md. Do not propose solutions — report findings only.
