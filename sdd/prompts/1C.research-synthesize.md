# SDD Phase 1C: Research Synthesis

This prompt combines findings from all research thread sub-agents into a single research.md.

---

You are synthesising research findings from multiple independent threads into a single research document. Each thread investigated a different aspect of the same problem in a separate sub-agent, without seeing the others' findings.

## Inputs

Read the problem definition:
- `docs/sdd/specs/{feature-name}/problem.md`

Read all thread outputs returned by the sub-agents.

## Your Job

1. **Organise thread findings** — transfer each thread's findings into the research document, preserving the independence of each thread's perspective.

2. **Identify conflicts** — where did different threads suggest contradictory approaches? These are real design decisions that the specification phase needs to resolve. Document both sides without picking a winner.

3. **Identify reinforcements** — where did multiple threads independently reach the same conclusion? These are high-confidence findings.

4. **Assess the problem definition** — this is critical. Given everything the research found, does the original problem definition in problem.md still hold? Consider:
   - Is the scope right, or did research reveal it should be bigger/smaller?
   - Are the stated constraints accurate, or did research uncover new ones?
   - Is the "What Good Looks Like" section still the right success definition?
   - Did research reveal the problem is actually a different problem than stated?

   If the problem definition needs updating, describe specifically what should change and why. The human must update problem.md before proceeding to specification. If the problem definition holds, state: "Problem definition confirmed — no reframe needed."

5. **Summarise** — 3-5 sentences on the most promising direction given all findings.

## Output

Produce the research document following the template in `docs/sdd/templates/research.md`. Save as `docs/sdd/specs/{feature-name}/research.md`.
