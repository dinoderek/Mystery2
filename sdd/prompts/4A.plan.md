# SDD Phase 4A: Task Breakdown Prompt

---

Read the following specification and produce a detailed task list for implementation.

## Mandatory Context Loading

Read these files:

- Specification: `docs/sdd/specs/{feature-name}/spec.md`
- `docs/architecture.md`
- `docs/game.md`
- `docs/project-structure.md`
- `docs/testing.md`

## Rules for Tasks

- Each task must be completable in a SINGLE, FRESH agent session
- Each task must have a clear, testable "done when" condition
- Tasks are ordered by dependency (what must exist before what)
- Tasks are as independent as possible — minimise cross-task coupling
- Each task references which spec sections it implements
- Each task specifies what files it creates or modifies
- Each task specifies which quality gates apply (`npm run lint`, `npm run typecheck`, `npm run test:unit -- --run`, etc.)
- Each task specifies which documentation needs updating (if any)
- If a task involves UI work, it must include component tests and/or Storybook stories
- If a task involves service-layer logic, it must include unit tests
- If a task involves database changes, it must include a migration file
- Group related tasks into phases

## Output

Produce a task breakdown following the structure in `docs/sdd/templates/tasks.md`. Save as `docs/sdd/specs/{feature-name}/tasks.md`.
