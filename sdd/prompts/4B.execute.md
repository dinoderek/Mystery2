## Rules

1. Implement ONLY what this task describes. Do not refactor unrelated code. Do not add features not in the spec.
2. Follow the constraints in the spec exactly. If the spec says "must not", then do not.
3. Write tests as specified in the task. Follow the test tiers in `docs/testing.md`.
4. Do not edit files outside the task's `Files affected` list. If the task cannot be completed safely without expanding scope, STOP and report the required change instead of guessing.
5. Update documentation listed in the task's "Doc updates" field.
6. If you encounter a blocker or ambiguity not covered by the spec, STOP and describe the issue. Do not guess.
7. Verify the task's `Done when` criteria before returning.
8. Run only the quality gates listed for this task, and report the results.
9. Do not commit changes or start another task. Return your results to the orchestrator.
10. After implementation, provide:
   - List of files created or modified
   - Test results from quality gates
   - Any issues the next task should be aware of
   - Documentation changes made
