# Data Model: Basic Authentication

**Feature**: 005-supabase-auth | **Date**: 2026-03-09

## Entities

### 1. User (Managed by Supabase Auth)

**Table**: `auth.users` (managed by Supabase — not directly modified)

| Field | Type | Notes |
|-------|------|-------|
| `id` | `uuid` (PK) | Auto-generated by Supabase Auth |
| `email` | `text` | Unique, used for login |
| `encrypted_password` | `text` | Managed by Supabase Auth |
| `created_at` | `timestamptz` | Auto |
| `updated_at` | `timestamptz` | Auto |

**Validation**: Email format enforced by Supabase Auth. Password minimum length configurable in `supabase/config.toml` (default: 6).

**Provisioning**: Accounts are created externally via:
- `supabase.auth.admin.createUser()` in testkit helpers
- SQL seed file for local dev (`supabase/seed/seed-auth-users.sql`)

### 2. Game Session (Modified)

**Table**: `game_sessions`

| Field | Type | Notes |
|-------|------|-------|
| `id` | `uuid` (PK) | Existing |
| `user_id` | `uuid` (FK → `auth.users.id`) | **NEW** — owner of this session |
| `blueprint_id` | `uuid` | Existing |
| `mode` | `text` | Existing |
| `current_location_id` | `text` | Existing |
| `current_talk_character_id` | `text` | Existing |
| `time_remaining` | `integer` | Existing |
| `discovered_clues` | `text[]` | Existing |
| `outcome` | `text` | Existing |
| `created_at` | `timestamptz` | Existing |
| `updated_at` | `timestamptz` | Existing |

**New RLS Policies** (replacing current `anon` policy):

```sql
-- Authenticated users can only see/modify their own sessions
create policy "Users can manage own sessions" on game_sessions
    for all
    to authenticated
    using (auth.uid() = user_id)
    with check (auth.uid() = user_id);
```

### 3. Game Event (Unmodified Schema)

**Table**: `game_events`

| Field | Type | Notes |
|-------|------|-------|
| `id` | `uuid` (PK) | Existing |
| `session_id` | `uuid` (FK → `game_sessions.id` ON DELETE CASCADE) | Existing |
| `sequence` | `integer` | Existing |
| `event_type` | `text` | Existing |
| `actor` | `text` | Existing |
| `payload` | `jsonb` | Existing |
| `narration` | `text` | Existing |
| `clues_revealed` | `text[]` | Existing |
| `created_at` | `timestamptz` | Existing |

**No schema changes** — ownership is inherited through `session_id` FK.

**New RLS Policy** (replacing current `anon` policy):

```sql
-- Authenticated users can only see/modify events for their own sessions
create policy "Users can manage own session events" on game_events
    for all
    to authenticated
    using (
        exists (
            select 1 from game_sessions
            where game_sessions.id = game_events.session_id
            and game_sessions.user_id = auth.uid()
        )
    )
    with check (
        exists (
            select 1 from game_sessions
            where game_sessions.id = game_events.session_id
            and game_sessions.user_id = auth.uid()
        )
    );
```

## Relationships

```text
auth.users (1) ──── (N) game_sessions (1) ──── (N) game_events
     │                        │                        │
     │  user_id FK            │  session_id FK         │
     │  ON DELETE: cascade*   │  ON DELETE: cascade    │
```

*Note: `auth.users` deletion cascading to `game_sessions` requires either a DB trigger or a `REFERENCES auth.users(id) ON DELETE CASCADE` on the FK. For pre-provisioned accounts that are rarely deleted, a trigger-based approach is preferred to avoid coupling to Supabase Auth internals.*

### Route Guard Logic

```text

  YES → Show terminal spinner

    NO  → Current route is /login?

      NO  → Save intended path, redirect to /login (FR-006)
```
